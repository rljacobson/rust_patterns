21## Information Hiding and Encapsulation in Rust
22
23Rust's language features enabling information hiding and encapsulation are much simpler
24and fewer in number than those in C++. In Rust, items and fields are private by default,
25and the visibility modifier `pub` is intentionally coarse-grained, limited to `pub(crate)`,
26`pub(super)`, and `pub(in path)`. There is no mechanism to grant selective access to
27private items. Rust has no concept of inheritance. Rust uses traits, which define
28a set of method signatures, to define shared interfaces. A trait can have one or more
29*trait constraints*, which look like superclasses if you squint, but which are really
30just assertions that an implementor of the trait must also implement some other trait.
31
32Rust's support for shared *implementation* is very limited. A trait can have a default
33implementation for a method, but even this support for shared implementation is hamstrung by the
34fact that traits cannot have data members of any visibility. Thus any default implementation
35can only assume about `self` what the trait itself asserts and cannot access any data directly.
36What's more, as a mechanism for defining public interfaces, trait methods are necessarily public.
37
38However, it turns out that it nonetheless is possible to achieve very fine-grained
39access and overridability restrictions on traits using a concept called _sealed traits_.
40
