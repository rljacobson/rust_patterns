1/*!
2`ValueVec<T>`: a by-value, `ref`-less vector with interior mutability.
3
4You can convert to and from a `Vec<T>` at zero cost.
5
6Key properties:
7- All mutating operations use `&self` (immutable receiver).
8- No references to elements are ever given out.
9- Elements are inserted/removed/moved **by value**.
10- Getting a value returns a `Clone` (or `Copy`) of the stored element.
11- Many shared immutable references to a `ValueVec` can exist simultaneously safely.
12
13Trade-offs:
14- You can’t borrow into the backing memory; you pay cloning cost to read.
15- In return, the backing storage may reallocate at will without invalidating
16  any external references (because none are ever issued).
17
18Functionality:
19
20For any functionality of `Vec<T>` that `ValueVec<T>` doesn't provide,
21you can use `into_vec` to convert to a `Vec<T>` at zero cost.
22
23Safety and Soundness:
24
25The fundamental requirement for soundness is that a mutable borrow of the internal `Vec` (or
26its owned memory) can never occur simultaneously with any other borrow. The only mutable borrows
27that exist are the temporary borrows in the method implementations we control. Therefore,
28`ValueVec` methods must not be re-entered while another method is still executing on the same
29instance. Doing so would create overlapping borrows of the internal `Vec` and cause undefined
30behavior. Our implementation makes no such explicit calls, so the only way this can happen
31is _implicitly_, in particular when the compiler inserts a call to the `V` type's `Drop`
32implementation. You would have to try pretty hard to trigger this, but it *is* possible.
33
34There are a few ways to deal with this requirement:
35
36- Require `V` to be `Copy`. `Copy` types are not allowed to have destructors. This is the simplest and easiest solution,
37  but it limits `ValueVec<V>` to a far more restrictive category of types than is necessary.
38- It _might_ be possible to carefully restrict the API of `ValueVec` and the implementation of its methods so that
39  re-entrancy does not cause UB, but this is very delicate and an open question.
40- Enforce a documented contract that `V` must not require destruction by wrapping all `V` values in
41  `std::mem::ManuallyDrop<V>`, which will prevent any destructor from running.
42- Prevent creation of a `ValueVec<V>` with an assertion of the form
43  `assert!(!needs_drop::<T>(), "Type must not have a destructor")`. (Perhaps used in combination with the strategy of
44  the previous bullet.)
45- Just make users pinky promise not to do weird things in the `Drop` implementation of `V`. This dispenses with
46  machine-checked correctness in favor of a human-checked contract, which may be unpalatable or even
47  prohibited in some settings, but which is probably good enough for the vast majority of use cases.
48
49
50# Options for `!Drop`
51
52A simple way to ensure soundness of `ValueVec<T>` is to require `T` to be `Copy`. But there are alternatives.
53
54### `std::mem::ManuallyDrop<T>`
55
56> A wrapper to inhibit the compiler from automatically calling T’s destructor. This wrapper is 0-cost.
57
58
59### Compiler error on `std::mem::needs_drop::<V>()`
60
61We can prevent construction of `ValueVec`s for values with destructors by using the `needs_drop` function:
62
63```rust
64impl<V> ValueVec<V> {
65    pub fn new() -> Self {
66        // Panics if `V` has a destructor.
67        assert!(!needs_drop::<V>(), "Type must not have a destructor");
68
69        ValueVec {
70            values: Vec::new(),
71        }
72    }
73}
74```
75
76## Homework assignments
77
78- Can we further constrain `V` to be `!Drop` (and/or `!Clone`) ?
79- Where is the compiler inserting `Drop`? Can we only implement mutating primitives such that
80  `Drop` is not called? (If we never "delete" something. Is `Drop` called when overwritten?)
81- If we provide a trivial implementation of `Drop` for all `Property` types, does that solve
82  the problem? (Could there be a hidden `Drop` impl called somehow by virtue of what the type is?)
83

86use std::{cell::UnsafeCell, fmt::Debug};

88/// A by-value, `ref`-less vector with interior mutability. Values of type `V` can be moved into and out of the vector.
89/// Cloning / copying getters exist if `V` implements `Clone`/`Copy`. See the module-level docs for important notes
90/// about soundness.
91pub struct ValueVec<V> {
92    data: UnsafeCell<Vec<V>>,
93}
94
95impl<V> ValueVec<V> {
96    /// Creates an empty `ValueVec`.
97    pub fn new() -> Self {
98        Self {
99            data: UnsafeCell::new(Vec::new()),
100        }
101    }
102
103    /// Creates with capacity.
104    pub fn with_capacity(cap: usize) -> Self {
105        Self {
106            data: UnsafeCell::new(Vec::with_capacity(cap)),
107        }
108    }
109
110    /// Current number of elements.
111    #[inline]
112    pub fn len(&self) -> usize {
113        // Safety: This is already an immutable operation
114        unsafe { (&*self.data.get()).len() }
115    }
116
117    /// Current capacity of the backing Vec.
118    #[inline]
119    pub fn capacity(&self) -> usize {
120        // Safety: This is already an immutable operation
121        unsafe { (&*self.data.get()).capacity() }
122    }
123
124    /// Returns true if the vector has no elements.
125    #[inline]
126    pub fn is_empty(&self) -> bool {
127        self.len() == 0
128    }
129
130    /// Ensures capacity for at least `additional` more elements.
131    pub fn reserve(&self, additional: usize) {
132        self.with_vec(|v| v.reserve(additional));
133    }
134
135    /// Shrinks the capacity as much as possible.
136    pub fn shrink_to_fit(&self) {
137        self.with_vec(|v| v.shrink_to_fit());
138    }
139
140    /// Pushes a value (by move) onto the end.
141    pub fn push(&self, value: V) {
142        self.with_vec(|v| v.push(value));
143    }
144
145    /// Pops and **returns** the last element (by move), or `None` if empty.
146    pub fn pop(&self) -> Option<V> {
147        self.with_vec(|v| v.pop())
148    }
149
150    /// Returns a **clone** of the element at `index`. Panics if `index` is out of bounds.
151    ///
152    /// This is a cloning version of [`ValueVec::at`].
153    pub fn get_cloned(&self, index: usize) -> Option<V>
154    where
155        V: Clone,
156    {
157        unsafe { (&*self.data.get()).get(index).cloned() }
158    }
159
160    /// Returns a **copy** of the element at `index` Panics if `index` is out of bounds.
161    ///
162    /// Use [`ValueVec::get_cloned`] for a cloning version of this method.
163    pub fn at(&self, index: usize) -> V
164    where
165        V: Copy,
166    {
167        unsafe { (&*self.data.get())[index] }
168    }
169
170    /// Moves a value into the slot at `index`, returning the old value (via move). Panics if `index` is out of bounds.
171    pub fn replace(&self, index: usize, value: V) -> V {
172        self.with_vec(|v| core::mem::replace(&mut v[index], value))
173    }
174
175    /// Sets the value of the slot at `index` to `value`. Panics if `index` is out of bounds.
176    pub fn set(&self, index: usize, value: V) {
177        self.with_vec(|v| v[index] = value)
178    }
179
180    /// Swaps the value at `index` with the provided one in place. Panics if `index` is out of bounds.
181    ///
182    /// The existing value ends up in `*value`.
183    pub fn swap_value(&self, index: usize, value: &mut V) {
184        self.with_vec(|v| {
185            core::mem::swap(&mut v[index], value);
186        })
187    }
188
189    /// Inserts `value` at position `index`, shifting elements to the right. Panics if `index` is out of bounds.
190    pub fn insert(&self, index: usize, value: V) {
191        self.with_vec(|v| {
192            v.insert(index, value);
193        })
194    }
195
196    /// Removes and returns the element at `index`, shifting elements left. Panics if `index` is out of bounds.
197    pub fn remove(&self, index: usize) -> V {
198        self.with_vec(|v| v.remove(index))
199    }
200
201    /// Removes and returns the element at `index` by swapping in the last element.
202    ///
203    /// O(1) removal when order does not matter.
204    pub fn swap_remove(&self, index: usize) -> V {
205        self.with_vec(|v| v.swap_remove(index))
206    }
207
208    /// Returns `true` if the `ValueVec` contains an element with the given value.
209    pub fn contains(&self, value: &V) -> bool
210    where
211        V: PartialEq,
212    {
213        self.with_vec(|v| v.contains(value))
214    }
215
216    /// Clears all elements.
217    pub fn clear(&self) {
218        self.with_vec(|v| v.clear());
219    }
220
221    /// Extends the vector by moving in elements from an iterator.
222    pub fn extend<I>(&self, iter: I)
223    where
224        I: IntoIterator<Item = V>,
225    {
226        self.with_vec(|v| v.extend(iter));
227    }
228
229    /// Returns a **snapshot** `Vec<V>` by cloning all elements.
230    ///
231    /// Use [`into`](ValueVec::into) or [`Vec::from`] to consume the `ValueVec`
232    /// and perform a zero-cost conversion without cloning.
233    pub fn to_vec(&self) -> Vec<V>
234    where
235        V: Clone,
236    {
237        unsafe { (&*self.data.get()).clone() }
238    }
239
240    /// Applies `f` with exclusive access to the inner Vec.
241    ///
242    /// **Safety:** `with_vec` temporarily obtains an exclusive `&mut Vec<V>`
243    /// from the internal `UnsafeCell` and passes it to the provided closure.
244    /// This is considered sound **only under controlled internal use**:
245    ///
246    /// - The mutable borrow does not escape the function.
247    /// - No references to elements are ever returned or stored; all access
248    ///   to elements occurs by value (move or clone).
249    /// - Only one mutable borrow of the internal `Vec` exists at a time,
250    ///   and it ends before the method returns.
251    ///
252    /// The one additional requirement for soundness is that the closure
253    /// passed to `with_vec` must not re-enter any other `ValueVec` method
254    /// (directly or indirectly) while it holds the mutable borrow. Such
255    /// re-entrancy could occur, for example, from a user-defined `Drop` or
256    /// `Clone` implementation of `V` that calls back into the same instance,
257    /// and would cause overlapping borrows of the internal `Vec`, leading to
258    /// undefined behavior.
259    ///
260    /// This function remains private to ensure that every closure passed to
261    /// it is under our control and has been manually verified to uphold the
262    /// no-reentrancy invariant. Its soundness depends on that internal
263    /// discipline.
264    #[inline]
265    fn with_vec<R>(&self, f: impl FnOnce(&mut Vec<V>) -> R) -> R {
266        // SAFETY: `UnsafeCell` permits obtaining a unique mutable reference here.
267        // We never let any references escape, and we serialize access by construction
268        // (each method call performs one short-lived exclusive borrow of the Vec).
269        let vec: &mut Vec<V> = unsafe { &mut *self.data.get() };
270        f(vec)
271    }
272}

274impl<V> Default for ValueVec<V> {
275    fn default() -> Self {
276        Self::new()
277    }
278}

280impl<V: Debug> Debug for ValueVec<V> {
281    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
282        // SAFETY: We create a temporary shared reference to the inner Vec.
283        // No mutable borrows of the Vec exist concurrently by design.
284        let vec = unsafe { &*self.data.get() };
285        vec.fmt(f)
286    }
287}

289impl<V> From<Vec<V>> for ValueVec<V> {
290    /// Wraps an existing `Vec` without copying its elements.
291    fn from(src: Vec<V>) -> Self {
292        Self {
293            data: UnsafeCell::new(src),
294        }
295    }
296}
297
298impl<V> From<ValueVec<V>> for Vec<V> {
299    fn from(val: ValueVec<V>) -> Self {
300        val.data.into_inner()
301    }
302}
303
304impl<V> IntoIterator for ValueVec<V> {
305    type Item = V;
306    type IntoIter = std::vec::IntoIter<V>;
307
308    fn into_iter(self) -> Self::IntoIter {
309        // SAFETY: We are consuming `self`, so there can be no remaining references
310        // to the inner Vec. It is therefore safe to move it out of the UnsafeCell.
311        let vec = self.data.into_inner();
312        vec.into_iter()
313    }
314}

321    fn push_pop() {
322        let v = ValueVec::new();
323        assert!(v.is_empty());
324        v.push(1);
325        v.push(2);

439    fn into_iterator_works_with_non_copy_types() {
440        #[derive(Debug, PartialEq, Eq)]
441        struct DropCounter<'a> {
442            val: i32,
443            drops: &'a std::cell::Cell<usize>,
444        }
445
446        impl<'a> Drop for DropCounter<'a> {
447            fn drop(&mut self) {
448                self.drops.set(self.drops.get() + 1);
449            }
450        }
451
452        let drops = std::cell::Cell::new(0);
453        {
454            let vv = ValueVec::from(vec![
455                DropCounter {
456                    val: 1,
457                    drops: &drops,
458                },
459                DropCounter {
460                    val: 2,
461                    drops: &drops,
462                },
463            ]);
464            let vals: Vec<_> = vv.into_iter().map(|c| c.val).collect();
465            assert_eq!(vals, vec![1, 2]);
466        }
467        // Both elements should have been dropped after iteration.
468        assert_eq!(drops.get(), 2);
